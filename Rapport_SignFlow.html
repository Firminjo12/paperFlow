7. Bugs Connus et En Cours de Résolution

  • React DOM : Crash insertBefore/removeChild sur certains composants React lors de démontages instables.
  • Traduction : Limitée à 500 caractères par requête (Bridage par l'API Externe Gratuite MyMemory).
  • Protéger PDF : Cette fonctionnalité dépend encore particulièrement du backend.
  • En cours : Les outils de censure (Censure) et de rognage (Crop) nécessitent encore des finitions visuelles.

9. Guide de Contribution (New Developers)

Bienvenue sur paperFlow ! 🚀 Pour maintenir la base de code propre et fonctionnelle, merci de suivre ces règles essentielles :

  1. Branches Isolées : Poussez toujours vos modifications sur une nouvelle branche dérivée de `main`. L'architecture est MERN, veillez à commit séparément les parties Frontend et Backend si possible.
  2. Zéro-Serveur en Priorité : L'un de nos mots d'ordre est la sécurité et les économies de serveurs. Si un outil PDF peut être développé 100% en local dans le navigateur grâce à pdf-lib, ne créez pas de route API sur le backend.
  3. Clean Code & UI : Réutilisez les composants UI déjà existants dans frontend/src/components/. Utilisez strictement TailwindCSS v4.
  4. Sécurité des Secrets : Toute clé asymétrique ou mot de passe doit passer par les fichiers .env. Ne jamais commit vos propres .env.

Une fois votre travail prêt, ouvrez une Pull Request descriptive expliquant vos solutions techniques (et tests menés).
